Claude v3 Additions — What Changed and Why
[ADDITION 1]
DriftSeverity Enum
Replaces raw uint8 (1, 2, 3) with a typed enum: MINOR / MODERATE / CRITICAL. Prevents invalid values at the type level and makes ABI calls self-documenting. Grok v2 used magic numbers — a common Solidity anti-pattern.
[ADDITION 2]
DRIFT_REPORTER_ROLE
In v2, any address could call logDrift() on any token — an open griefing / spam attack vector. v3 requires callers to hold the DRIFT_REPORTER_ROLE or be the token holder for MINOR/MODERATE. CRITICAL remains owner-only.
[ADDITION 3]
commitmentHash (ZK Anchor)
A bytes32 slot added to IdentityRecord for future ZK-SNARK selective disclosure. Pass bytes32(0) on mint if not yet used. This future-proofs the record for Groth16/PLONK proofs without a schema migration.
[ADDITION 4]
whenNotPaused on bindIdentity
In v2, the pause modifier was applied to logDrift but NOT to mintBirthRight — meaning minting continued during an emergency pause. v3 applies whenNotPaused to bindIdentity, closing the gap.
[ADDITION 5]
Safe ETH via call{}
v2 used transfer() which has a 2300-gas stipend. This fails silently when the recipient is a smart-contract wallet (Gnosis Safe, etc.). v3 uses call{value:...}("") with explicit require() checks — the current Solidity best practice.
[ADDITION 6]
getDriftsByToken()
v2 only exposed a flat public driftLog[] array. To find events for a specific token, callers had to iterate the entire array off-chain. v3 maintains a per-token index mapping and a getDriftsByToken() view function.
[ADDITION 7]
Ownable2Step
Replaces Ownable with Ownable2Step from OpenZeppelin v5. Requires the new owner to explicitly accept before the transfer completes. Prevents accidental ownership transfer to a wrong or zero address.
[ADDITION 8]
_tokenExists() Correction
v2 implemented a custom _exists() comparing tokenId to the counter — fragile with gaps. v3 uses the same pattern but properly via the counter, and will migrate to ownerOf() revert-check when OZ v5 fully deprecates _exists().
