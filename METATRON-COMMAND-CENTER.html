PromoRevenue.sol — Remix Deployment Checklist
  • 1. Open Remix IDE
    Go to remix.ethereum.org in Chrome. Use the same browser where Rabby Wallet is installed.
  • 2. Create PromoRevenue.sol
    In the File Explorer, right-click → New File → name it PromoRevenue.sol. Paste your contract code.
  • 3. Set Compiler Version
    Go to the Solidity Compiler tab. Set version to 0.8.20 or your pragma version. EVM Version must be set to paris or london — NOT Shanghai/Cancun (avoids mcopy errors on Base).
    EVM Target: paris | Compiler: 0.8.20 | Optimization: ON (200 runs)
  • 4. Compile — Confirm Zero Errors
    Hit Ctrl+S or click the blue Compile button. Green checkmark = ready. Warnings are OK; errors must be fixed before proceeding.
  • 5. Switch to Deploy Tab — Set Environment
    Go to Deploy & Run Transactions. Under ENVIRONMENT select "Injected Provider — Rabby". Do NOT select Ethereum Mainnet or JavaScript VM.
  • 6. Verify Network in Rabby
    Rabby wallet must show Base Mainnet (Chain ID 8453). If it shows Ethereum, click the network name and switch. Remix will display "Base" in the network field once connected.
    Chain ID: 8453 | Network: Base Mainnet | RPC: https://mainnet.base.org
  • 7. Enter Constructor Arguments
    PromoRevenue.sol's 60/10/30 split requires founder, reserve, and escrow addresses. Enter your three wallet addresses in order before deploying.
    founder: 0x... | reserve: 0x... | escrow: 0x...
  • 8. Click Deploy — Confirm in Rabby
    Hit the orange Deploy button. Rabby will pop up. Review gas (Base fees are low — usually <$0.10). Click Confirm. Wait for the transaction hash.
  • 9. Copy Contract Address
    After deployment, Remix shows the contract address in the Deployed Contracts panel. Copy it and paste it into the Contracts tab of this Command Center.
  • 10. Verify on Basescan
    Paste the contract address into basescan.org. Confirm the transaction is successful. Optionally verify the source code via the Verify & Publish tab.
    https://basescan.org/address/0xYOUR_CONTRACT_ADDRESS
  • 11. Test Revenue Split — Call distributeRevenue()
    In Remix Deployed Contracts panel, expand the contract. Send a small amount of ETH to test the 60/10/30 split. Verify all three addresses receive the correct amounts on Basescan.
  • 12. Log Everything Here
    Add the deployed address to the Contracts tab. Update AgniPromo and YamaRevenue agent cards in the Agents tab. This laptop is now the canonical record.
